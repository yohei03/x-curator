## 懸念点

### cloud_infra カバレッジ不足 (約10%)
**detail**: Cloud Run 検索で意味的に無関係な投稿（宗教的"cloud"、野球チーム名、クラウドホスティング広告スパム）が約60%を占めた。有効取得は実質8件程度。
**hint**: "Cloud Run" は X での検索クエリとして曖昧すぎる。Google Cloud の固有技術名（"Vertex AI"、"GKE"、"GCP"）や "kubernetes infra" 等への変更が考えられる。

### ノイズ率超過 (約25%, 目標20%)
**detail**: Cloud Run 検索が主因（12/20件が無関係）。ClawHost 系スパム返信も LLM agent・Cloud Run 検索に散見された。
**hint**: Cloud Run クエリの見直しが最大のインパクト。スパムアカウント対策として `-filter:links` や `min_faves:2` 等の絞り込み追加も有効な可能性がある。

### 一次情報率目標未達 (約15%, 目標20%)
**detail**: 検索ソースは二次報道・解説記事が多く、著名人の自社実験・当事者発信が少ない。author 層（simonw・GergelyOrosz）は質が高いが量が少ない（合計20件）。
**hint**: author_seeds の追加（例：@levelsio は今日 870いいね のVPS記事を投稿）、もしくは per-author の max_results を 20 に増やすことで一次情報率が改善する可能性がある。

### karpathy 空振り
**detail**: 24時間ウィンドウ内に投稿なし（from:karpathy で0件）。
**hint**: 投稿頻度が低い著者は searchPostsRecent の24h窓で空振りが発生しやすい。getUsersPosts を直接呼ぶか、ウィンドウを72hに拡大する運用が必要。
